# 📊 Hackathon Evaluation Criteria

This document details the mandatory requirements, daily scoring criteria, pitch regulations, and submission guidelines for the hackathon.

---

## 🛑 Mandatory Elements (Eligibility for the Pitch)

To be allowed to pitch your project, the following are strictly required:

- **Daily Artifacts:** Submissions from each day of the hackathon must be provided.
- **Team Composition:** List of team members and their responsibilities (ideally in a diagram form, e.g., using `ng-diagram`, though this is not a hard requirement).
- **Public Accessibility:** All links submitted in the final form must be publicly accessible. **If the judges cannot access your links, the team will not be evaluated.**

---

## 📅 Daily Milestones & Scoring [100 Points Total]

### Day 1: Business Process Analysis [20p]

- **Deliverable:** Business analysis of the process using BPMN or Event Storming.
- **Format:** Presented via a Miro board, another similar interactive tool, or a clear, readable screenshot.
- **Submission:** A link must be provided in the submission form.

### Day 2: Design System & Tokens [20p]

- **Deliverable:** Correct application of the Design System, utilizing Design Tokens (colors, typography, spacing, radii, shadows, etc.).
- **Requirement:** Maintain user interface consistency across all screens.

### Day 3: Working Codebase [20p]

- **Deliverable:** Link to a Git repository with the working application code.
- **Requirement:** Clear instructions in the README on how to run the project locally.

### Day 4: Evaluation & Metrics Report [20p]

- **Deliverable:** Link to an evaluation report.
- **Requirement:** The report must cover the path from the list of input scenarios to the idea evaluation engine and resulting metrics.

### Day 5: Deployment [20p]

- **Deliverable:** Link to a publicly hosted, deployed, and fully operational version of the application.

---

## 🎤 Pitch Rules & Guidelines [25 Points]

- **Slides:** Presentation slides are highly recommended (though not strictly mandatory).
- **Language:** English is highly suggested for the presentation and is at least recommended for the slides (as there will be an English-speaking judge on the panel).
- **Duration:** Exactly **5 minutes**. A visible countdown timer will be running, and speakers will be interrupted mid-sentence immediately when the timer runs out.
- **Questions:** There will be **no Q&A session** with the jury.

---

## 📝 Submission Form Fields

When submitting your final project, you must provide:

1. Link to the Day 1 artifact (Miro board / BPMN).
2. Link to the Figma design files.
3. Link to the code repository.
4. Link to the Day 4 evaluation report.
5. Link to the live, working application.
6. *(Optional)* Link to the pitch presentation slides.

---

## 👥 Peer Team Ratings [25 Points]

In addition to the jury evaluation, teams are required to rate the presentations of all other participating teams.

- **Rule:** You will receive these **25 points** only if at least one member of your team submits ratings for **ALL** other teams.
